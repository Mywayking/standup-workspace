# V0 主链路验收清单

## 一、项目目标

本阶段只做一件事：

让用户在 `/write` 页面，从一个生活素材出发，完整走完：

素材输入 → 提炼前提 → 选择前提 → 找角度 → 选择角度 → 生成草稿 → 改稿 → 保存历史 → 刷新恢复。

本阶段不新增复杂功能，不优化非核心页面，不做额外 Tab，不做复杂模型配置。

## 二、核心用户故事

作为一名脱口秀创作者，我希望打开 `/write` 后，可以输入一个真实生活素材，并在 AI 的引导下，一步步生成可继续打磨的段子草稿。

## 三、必须通过的主流程

### 1. 打开页面

访问：

`https://standup.alwayshaha.art/write`

用户应该看到一个清晰的创作入口，而不是多个混乱面板。

验收标准：

- 页面无报错
- 桌面端布局正常
- 移动端布局正常
- 不出现旧版组件和新版组件同时渲染
- 输入框可见
- 主按钮可见

### 2. 输入素材

用户输入：

> 我外公特别偏心，给我表哥买自行车，不给我买，他说我是外孙，是外面的孙子。

验收标准：

- 输入不卡顿
- 移动端键盘弹起后输入框不被遮挡
- 发送按钮可点击
- 空输入时不能提交，并有提示

### 3. 提炼前提

点击"提炼前提"。

验收标准：

- 前端发起请求
- 后端收到请求
- AI 开始生成
- 页面有 loading / streaming 状态
- 生成失败时有错误提示
- 成功后出现前提卡片
- 前提卡片内容不为空

### 4. 选择前提

用户点击某一条前提继续。

验收标准：

- 被选中的前提有明确状态
- 下一步按钮变成"找角度"
- sourceCardId 正确记录
- 不需要用户手动复制内容

### 5. 找角度

点击"找角度"。

验收标准：

- 请求内容来自被选中的前提
- 成功后生成角度卡片
- 至少返回 3 个角度
- 每个角度有标题和简短解释
- 失败时可以重试

### 6. 选择角度

用户点击某一条角度继续。

验收标准：

- 被选中的角度有明确状态
- 下一步按钮变成"生成草稿"
- sourceCardId 正确记录
- 上下文不丢失

### 7. 生成草稿

点击"生成草稿"。

验收标准：

- 请求内容来自选中的角度，同时携带原始素材和前提
- 成功后生成草稿卡片
- 草稿像脱口秀口语稿，不是普通文章
- 内容可复制
- 内容可继续改稿

### 8. 改稿

点击"改稿"。

验收标准：

- 当前草稿能自动作为改稿输入
- 用户可以追加改稿要求
- 成功后生成新版草稿
- 旧版不丢失
- 新版和旧版有来源关系

### 9. 保存历史

验收标准：

- 每一步生成的卡片都被保存
- 页面侧边栏或抽屉能看到作品脉络
- 用户知道当前内容来自哪一步
- 刷新页面后内容仍然存在

### 10. 移动端验收

在 390px 宽度下验收。

验收标准：

- 页面不横向溢出
- 卡片宽度正常
- 输入框固定在可用位置
- 键盘弹起后不遮挡输入
- 快捷按钮不过度拥挤
- 历史 / 作品脉络进入抽屉
- 当前主操作清晰

## 四、本阶段不做

以下功能本阶段不做，避免范围失控：

- 多 Tab 全量重构
- 用户自定义 Key
- 多模型管理页面
- 复杂作品库
- 专场管理
- 社区分享
- 海报生成
- 复杂数据分析
- 大量 Quick Tools

## 五、P0 Bug 定义

出现以下任一问题，即 V0 不通过：

- 页面打不开
- 输入后按钮无反应
- AI 请求失败但无提示
- 一直 loading 无法恢复
- 生成结果为空
- 选择前提 / 角度后上下文丢失
- 改稿没有带入当前草稿
- 刷新后历史丢失
- 移动端严重错位
- 新旧组件同时显示

## 六、完成标准

V0 完成的标准：

用户可以在桌面端和移动端，从一个素材开始，连续完成：

素材 → 前提 → 角度 → 草稿 → 改稿

并且所有结果可保存、可恢复、可继续加工。
