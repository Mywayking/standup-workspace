## 三、真实用户体验问题清单

### 🔴 P0 — 阻断真实使用

**无 P0 问题。** 3 条素材均完整走通，无阻断。

---

### 🟡 P1 — 明显体验问题

#### P1-1：创作进度面板 step 计数器不更新

**位置**：桌面端右侧「创作进度」面板
**现象**：当前显示"桌面常驻 · **0/6 步**"，但用户已完成素材→前提→角度→草稿至少 4 步。
**影响**：用户不知道自己在创作流程中走了多远，无法判断下一步该做什么。
**证据**（从测试错误上下文提取）：
```
1) <h2 class="text-[17px] font-semibold text-[#25231F]">创作进度</h2>
   aka locator('aside').filter({ hasText: '作品脉络×创作进度桌面常驻 · 0/6 步' })
```
**最小修复**：让 `currentStep` 正确反映当前卡片的类型，更新右侧步骤指示器的 `currentIdx`。

---

#### P1-2：「更毒舌」按钮意图不明确

**位置**：premise 卡上的 action 按钮组
**现象**：按钮文案为"更毒舌"，用户不清楚点击后会发生什么。
**影响**：用户可能不敢点，或者点错了期望得到不同的结果。
**对比**：同组其他按钮文案"找角度"、"扩成草稿"都很明确。
**最小修复**：将"更毒舌"改为更具体的操作描述，如"🔄 继续改稿"或"📝 继续打磨"。
**建议文案**：参考"找角度"/"扩成草稿"的命名模式，改为"继续打磨"或"再锐一点"。

---

#### P1-3：卡片来源关系感知弱

**位置**：卡片底部「来源：用户输入 → premise → angle → rewrite」
**现象**：
- 「来源」标签文字小（11px），颜色浅（`#C5BAAA`），容易被忽略
- 移动端已隐藏（`hidden md:block`），用户完全看不到来源关系
**影响**：用户改稿后不知道新版草稿和旧版的关系，无法追溯。
**最小修复**：
1. 在桌面端把「来源」字号适当放大（12-13px），颜色加深
2. 在移动端（md 以下）也显示，但可以简化，如"来自「角度」"

---

### 🟢 P2 — 体验优化建议

#### P2-1：premise 卡上的 action 按钮过多

**位置**：提炼完前提后，premise 卡出现 8 个 action 按钮
**现象**：用户看到一堆按钮，不知道该点哪一个。
**建议**：
- 每个 premise 卡只展示最相关的 2-3 个 action（如"找角度"、"扩成草稿"）
- 其他 action 通过"更多"菜单展开
- 或者按「下一步」逻辑，只显示当前最该点的那个按钮

#### P2-2：input placeholder 文案较长

**当前文案**："写下一个素材、一个念头，或者一句还没成熟的梗。"
**问题**：句子较长，用户第一次可能不知道该写什么。
**建议**：简化为"讲讲你的生活故事…"或"输入你的脱口秀素材"。

#### P2-3：「来源」标签文字技术化

**当前文案**："来源：用户输入 → premise → angle → rewrite"
**问题**："premise"、"rewrite" 是内部术语，用户不一定理解。
**建议**：改为更友好的表述，如"来自：素材 → 前提 → 角度 → 草稿"，或"脉络：前提 → 角度 → 草稿"。

#### P2-4：移动端改稿后不知道下一步

**位置**：移动端生成草稿后
**现象**：生成草稿后，输入框在底部，但没有明确提示用户可以做什么。
**建议**：在草稿卡片下方显示一句引导文案，如"点击下方「改稿」继续打磨"。

#### P2-5：桌面端左侧栏 session 标题为空时显示「」

**位置**：桌面端左侧作品列表
**现象**：session 未填写标题时，列表项显示为空或「」。
**建议**：未命名 session 至少显示素材前 10 个字作为默认标题。

## 五、结论

### 总体评价

| 维度 | 评价 |
|------|------|
| 功能完整性 | ✅ 核心链路 3/3 素材全部走通 |

**否。** 用户可以在不知道上述问题的情况下完成一次完整创作。

**建议进入 ✅**，但建议优先修复 P1-1（步骤计数器）和 P1-3（来源感知）。
