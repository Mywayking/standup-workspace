# standup-workspace 优化方案

> 来源：大会长产品优化建议
> 时间：2026-04-28

## 1. 优化目标

让用户在手机和桌面端都能清楚知道：
- 我现在在哪一步
- 刚刚生成了什么
- 下一步该点哪里
- 历史内容保存在哪里

**核心心智**：从"AI 工具页" → "脱口秀创作者的写作工作台"

## 2. 信息架构

### 桌面端
```
┌─────────────────────────────────────────────┐
│ 顶部导航：Logo / 当前会话 / 保存状态 / 登录 │
├──────────────┬────────────────┬─────────────┤
│ 左侧历史区 │ 中间创作区 │ 右侧辅助区 │
│ 会话列表   │ 输入+结果卡片│ 技法/提示/设置│
│ 素材库     │ 主流程时间线 │ 模型状态    │
└──────────────┴────────────────┴─────────────┘
```

### 移动端
```
┌──────────────────────┐
│ 顶部：当前步骤 + 保存状态 │
├──────────────────────┤
│ 流程条：素材→前提→角度→改稿 │
├──────────────────────┤
│ 结果卡片时间线 │
├──────────────────────┤
│ 输入框 │
├──────────────────────┤
│ 底部主按钮 │
└──────────────────────┘
```

## 3. 核心工作流优化

### 卡片化输出原则

每张 AI 输出卡片必须包含：
1. **标题**：这是什么类型的输出
2. **内容**：AI 生成的实际内容
3. **解释**：为什么这样写（可选）
4. **下一步推荐动作**：一个主按钮
5. **可选动作**：复制/保存/更多

### 按钮降噪原则

每张卡只保留一个主按钮：
```
前提卡：
主按钮：继续找角度
更多：换一个前提 / 复制 / 删除 / 收藏

角度卡：
主按钮：选中并改稿
更多：展开解释 / 再生成 / 收藏

改稿卡：
主按钮：继续优化
更多：复制 / 保存作品 / 生成标题 / 删除
```

## 4. 历史记录优化

### 来源链路

每张卡底部显示：
```
来源：素材《程序员加班》 → 前提 2 → 角度 1
```

### 保存反馈

所有保存动作必须有即时反馈：
```
✅ 已保存到本机
✅ 已同步到云端
登录后可跨设备同步
❌ 保存失败，点击重试
```

## 5. AI 流式生成优化

### 统一状态机

```typescript
type StreamingStatus =
  | "idle"
  | "connecting"
  | "streaming"
  | "success"
  | "error"
  | "timeout"
  | "cancelled";
```

### 生成中文案

```
正在理解素材...
正在提炼冲突...
正在生成前提...
正在整理结果...
```

超过 15 秒：
```
这次生成有点慢，可以继续等待或取消重试
[继续等待] [取消]
```

### 模型信息展示

```
由 GLM-5 生成 · 4.2s · 已自动保存
DeepSeek 超时，已切换到 MiniMax · 8.7s
```

## 6. 移动端优化

### 核心原则

移动端用户场景：
- 地铁上记灵感
- 开放麦后改一句
- 临睡前整理素材
- 演出前快速看稿

优先级：**快速输入 > 快速保存 > 清楚下一步 > 方便复制**

### 底部输入栏

```
┌────────────────────┐
│ 写点素材、前提或草稿... │
├────────────────────┤
│ [素材] [前提] [梗] [草稿] │
│ [开始创作] │
└────────────────────┘
```

底部输入框支持自动识别：
- 输入较长生活经历 → 默认提炼前提
- 输入一句 punchline → 默认倒推前提
- 输入已有段子 → 默认改稿

## 7. UI 视觉优化

### 风格关键词

```
白底 / 浅灰分区 / 圆角卡片 / 低饱和主色 / 少边框 / 强留白 / 明确主按钮
```

### 颜色建议

```
背景：#F7F7F8
卡片：#FFFFFF
主色：#2563EB 或 #111827
辅助色：#6B7280
成功：#16A34A
错误：#DC2626
警告：#F59E0B
```

### 卡片层级

```
普通卡片：白底 + 1px 浅边框 + 12px 圆角
最新卡片：浅蓝底 + "最新"标签
收藏卡片：右上角星标
失败卡片：浅红底 + 重试按钮
```

## 8. 账户设置优化

### 设置页结构

```
/settings/profile

基础资料：昵称/头像/简介/常写题材/喜剧风格
账号安全：手机号/邮箱/修改密码
模型设置：系统模型 vs 用户 API Key
数据管理：导出作品/清空本地/同步状态
```

### 用户 Key 逻辑

```
没有填写 Key：使用系统 TokenHub
填写了 Key：优先使用用户自己的 Key
用户 Key 失败：提示用户，不自动消耗系统 Key
```

## 9. 后端优化

### 统一接口

```
POST /api/write/tasks
GET /api/write/tasks/{task_id}/stream
GET /api/write/sessions
POST /api/write/sessions
GET /api/write/sessions/{id}
POST /api/write/cards
PATCH /api/write/cards/{id}
DELETE /api/write/cards/{id}
```

### 统一模型调用返回

```typescript
LLMResult {
  content: string
  provider: string
  model: string
  latencyMs: number
  fallbackUsed: boolean
  error?: string
}
```

## 10. 前端状态管理

三层分离：

```
UI State：当前选中的 tab、弹窗、抽屉、loading
  → React State

Workflow State：当前 session、cards、当前输入、当前步骤
  → Zustand

Server State：用户资料、云端历史、模型配置
  → TanStack Query
```

## 11. 迭代拆分

### Phase 1：稳定性修复

```
1. 修复 build/typecheck/lint
2. 统一 useStreamingTask
3. 确认后端 stream 接口真实可用
4. 修复按钮点击无反馈
5. 修复移动端横向溢出
6. 修复历史记录保存感知
```

### Phase 2：工作台体验升级

```
1. 卡片化输出
2. 来源链路展示
3. 会话列表
4. 收藏功能
5. 底部输入栏
6. 移动端历史抽屉
```

### Phase 3：账户与模型设置

```
1. 完善账户资料
2. 修改密码
3. 用户 API Key
4. 模型选择
5. 模型可用性测试
6. 生成记录展示模型信息
```

### Phase 4：创作者高级能力

```
1. 作品集
2. 开放麦版本管理
3. 一稿多版本对比
4. 表演提示
5. 段子结构分析
6. 喜剧技法解释
7. 根据个人风格改稿
```

## 12. 优先级排序

```
第一优先级：稳定生成链路
第二优先级：移动端体验
第三优先级：历史记录感知
第四优先级：账户和模型设置
第五优先级：高级创作能力
```
