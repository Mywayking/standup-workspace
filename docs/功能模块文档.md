## 一、产品定位

**喜剧分析工作台**是一个 AI 驱动的单口喜剧创作工具。用户输入脱口秀段子文本，AI 自动分析结构、节奏、技巧，输出专业拉片报告。同时支持脱口秀写作、单口喜剧编剧、段子诊断等场景。

**核心用户**：脱口秀创作者、喜剧编剧、段子爱好者

## 三、详细功能说明

### 1. 创作工作台 `/write`

**入口**：`https://standup.alwayshaha.art/write`

#### 1.1 提炼前提
- **功能**：输入一段生活素材（事件/观察/情绪），AI 提炼可上台说的喜剧前提
- **输入**：一段文字（建议 20 字以上）
- **输出**：
  - 主题识别
  - 态度判断
  - 核心矛盾
  - 5 个前提候选
  - 推荐前提
  - 场景建议
- **模型**：`deepseek-v4-pro`（thinking disabled）
- **耗时**：约 13 秒
- **接口**：`POST /api/write/premise/stream`

#### 1.2 梗写前提
- **功能**：输入一句梗/笑点/吐槽，反推生成可用的喜剧前提
- **输入**：一句话（一个笑点/吐槽/灵感）
- **输出**：反推主题 + 3-5 个前提候选
- **接口**：`POST /api/write/joke-to-premise/stream`

#### 1.3 找角度
- **功能**：输入已有前提/判断/结论，AI 帮你找切入点和表演角度
- **输入**：前提文本
- **输出**：
  - 当前问题诊断
  - 角度列表（3-5 个）
  - 推荐角度 + 理由
  - 可改写方向
- **接口**：`POST /api/write/find-angles/stream`

#### 1.4 改稿
- **功能**：输入段子草稿，AI 给出诊断 + 修改建议 + 改写版本
- **输入**：段子完整文本
- **输出**：
  - 原文诊断（问题分析）
  - 优先修改问题（3 个）
  - 改写版本
  - 可替换句子
  - 技巧标签
- **接口**：`POST /api/write/rewrite/stream`

#### 1.5 创作历史记录

**存储策略**：
| 状态 | 存储方式 | 提示文案 |
|------|---------|---------|
| 未登录 | localStorage | 已保存到本机浏览器，登录后可跨设备保存 |
| 已登录 | localStorage + 后端云端 | 已保存到你的创作会话 |

**保存上限**：本地 30 条会话

#### 1.6 云端同步（登录用户）
- **后端接口**：`/api/workflow-sessions` 系列
- **表结构**：`workflow_sessions` + `workflow_cards`

### 3. 个人设置 `/settings/profile`

**入口**：`https://standup.alwayshaha.art/settings/profile`

#### 3.1 账户资料
- **接口**：`GET/PUT /api/profile/me`
- **字段**：nickname / username / avatar_url / bio
- **行为**：首次访问自动创建默认 profile

#### 3.2 认证
- **注册**：`POST /api/auth/register`
  - 参数：`identifier`（邮箱）、`identifierType`（"email"）、`password`、`nickname`
  - Cookie：`access_token`（30 天有效期，HttpOnly，SameSite=Lax）
  - 生产环境自动带 Secure flag

- **登录**：`POST /api/auth/login`
  - 参数：`identifier` + `identifierType` + `password`
  - 失败返回：401 "账号或密码错误"

- **登出**：`POST /api/auth/logout`
  - 清除 `access_token` Cookie

- **当前用户**：`GET /api/auth/me`
  - 返回：id / display_name / email / identifier

## 五、技术架构

### 前端
- **框架**：Next.js 15 (App Router)
- **状态管理**：React Context + localStorage
- **样式**：Tailwind CSS
- **构建**：TypeScript

### 后端
- **框架**：FastAPI
- **数据库**：SQLite（`backend/data/standup.db`）
- **ORM**：SQLAlchemy
- **模型**：deepseek-v4-pro（主力）+ MiniMax M2.7（备用）

### LLM 输出解析
- `backend/app/llm/parsers/json_repair.py`
  - 三层策略：直接解析 → 提取 JSON → 修复后解析
  - 修复中文标点（，。：→ ,:")
  - 修复尾部逗号
- 解析失败返回 SSE `event: error`，友好提示用户重试

### 认证机制
- Cookie：`access_token`（HttpOnly，30 天）
- 生产环境自动带 Secure flag（`ENV=production`）
